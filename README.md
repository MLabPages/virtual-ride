# バーチャルライド (virtual-ride)

ルームバイクを漕ぐと、街並みの映像が**速度に連動して**流れる web アプリです。
退屈しがちな室内トレーニングを、観光気分で楽しめるようにします。

公開URL: https://mlabpages.github.io/virtual-ride/

## できること

- **速度連動の映像再生** — 速く漕ぐほど映像も速く流れ、止まると映像も止まります
- **2つの計測方法**
  - 📷 **カメラ計測** — スマホをバイクの横に置くと、ペダル(足元)の周期的な動きを
    検出して回転数(rpm)→速度に換算します(追加機器なし)
  - 🎚 **手動** — 画面のスライダーで速度を設定します
- **PC・モニター表示** — PC や大きいモニターで `display.html` を開き、
  スマホはペダル撮影専用として使えます。接続前でも「18 km/hで試運転」で表示を確認できます
- **前に進む景色を「旅」のように連続再生** — 静かな街並みから田舎道・森・海辺の町・山あいの道へつなぎ、朝焼けで締める8シーンのルート(1周約3分強・18km/h想定、周回カウントつき)。全映像を1本ずつ検査し、走行視点でない素材(空撮・固定カメラ・海を横から眺めるだけの映像)を除外。各映像の「動きの量」を計測して、同じ速度なら同じ体感になるよう再生倍率を揃えています
- **旅の現在地が分かるHUD** — 現在の景色、ルート進捗、次の景色、現在速度での残り時間を表示します
- **走行記録** — 距離、時間、平均速度、最高速度を端末内に保存し、12時間以内の再読み込みなら続きから再開します
- **没入表示** — 走行中は操作バーを自動で隠し、タップ・マウス・キー操作で戻せます
- 速度連動の風音(任意・端末内で生成)、画面消灯の防止、全画面表示、キーボード操作

## プライバシー・セキュリティ

- **カメラ映像は端末のブラウザ内だけで処理し、保存も送信もしません。**
  PC・モニター表示に送る場合も、送信されるのは速度などの数値だけです。
- PC・モニター表示の接続仲介には PeerJS を使います。カメラ映像は送りません。
- 設定と走行記録は端末の localStorage にのみ保存されます。走行記録は12時間で自動的に新しいセッションになります。
- 通信が HTTPS になるため、カメラ機能は公開URL(GitHub Pages)上で正しく動作します。

## 使い方(スマホ)

1. PC・モニターで `https://mlabpages.github.io/virtual-ride/display.html` を開く
2. スマホで上記の公開URLを開き、🖥ボタンから表示画面の4桁コードを入力する
3. スマホで「📷 カメラ計測」を選び、カメラの使用を許可する
4. スマホをバイクの横に固定し、ペダル(足元)だけが動いて見える角度にする
5. 漕ぎ始めると、PC・モニター側の映像が速度に合わせて流れます

手動モードでは「18 km/hで出発」またはスライダーですぐ試せます。キーボードでは
`Space` で開始・停止、矢印キーで速度変更、`F` で全画面、`R` で走行記録を開きます。

うまく検出できないときは、スマホを固定してから、⚙ 設定で「検出感度」や
「ギアの重さ」「足の映り方」を調整してください。机や背景など画面全体が揺れる構図は
誤検出しやすいので、ペダルだけが周期的に動く構図にします。

## 開発メモ

ビルド不要の静的サイトです(`index.html` / `app.js` / `styles.css` / `videos/`)。
公開前の構文・ルート定義・HTML参照チェックは `node verify.mjs` で実行できます。
`main` ブランチへの push で GitHub Pages に自動デプロイされます
(`.github/workflows/pages.yml`)。
